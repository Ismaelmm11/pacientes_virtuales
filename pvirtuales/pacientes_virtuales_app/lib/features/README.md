# Directorio `features`

Este directorio es el núcleo de la arquitectura **Feature-Driven**.

Cada subdirectorio aquí representa una funcionalidad autocontenida de la aplicación (ej. `auth`, `patient_chat`, `profile`). Agrupar el código por funcionalidad en lugar de por tipo (ej. todas las pantallas juntas) hace que la aplicación sea mucho más fácil de escalar y mantener.
